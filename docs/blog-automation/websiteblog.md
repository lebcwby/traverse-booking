## Post Writing Instructions (9-Step SEO Framework)

For every post, Claude Code must follow these steps:

### Step 1 — Requirements
Confirm: primary keyword, secondary keywords, word count (1,400–1,800), content type, audience intent, CTA goal

### Step 2 — Research
Web search for current facts, dates, events before writing. Never invent statistics.

### Step 3 — Title
- ≤60 characters
- Primary keyword front-loaded
- Power word or year where relevant

### Step 4 — Meta Description
- 150-160 characters
- Primary keyword included
- Implied CTA

### Step 5 — Structure
H1 → intro (hook + promise + primary keyword in first 100 words) → H2 sections → H3 sub-topics → FAQ → conclusion with CTA

### Step 6 — Write
- Natural keyword integration
- Concrete specifics and data-backed claims
- Write for humans first, search engines second
- Use markdown: ## for H2, ### for H3, **bold**, [text](url) for ALL links

### Step 7 — On-Page SEO
- Keyword density ~1-2%
- 2-5 internal links using proper markdown [text](url) format
- 2-3 authoritative external links
- FAQ section: minimum 5 questions, answers 40-60 words each

### Step 8 — Internal Links
Always include links to relevant booktraverse.com pages and sub-brand sites

### Step 9 — Final Review
Score against: keyword placement, title tag, meta description, header hierarchy, readability, internal links, external links, FAQ presence, content depth, search intent match

## Post Format for Supabase

Each post should be inserted into Supabase with these fields:

```json
{
  "title": "Post title here",
  "slug": "post-title-slug-here",
  "meta_description": "150-160 character meta description",
  "category": "Travel Guides | Owner Resources | Travel Tips",
  "tags": ["tag1", "tag2", "tag3"],
  "content": "<full HTML content>",
  "status": "draft",
  "published_at": "2026-05-12T08:00:00Z",
  "author": "Traverse Hospitality",
  "featured_image": null
}
```

### Markdown to HTML Conversion Rules
- `## Heading` → `<h2>Heading</h2>`
- `### Heading` → `<h3>Heading</h3>`
- `**bold**` → `<strong>bold</strong>`
- `[text](url)` → `<a href="url">text</a>`
- `- item` → `<ul><li>item</li></ul>`
- `---` → `<hr>`
- Regular paragraph → `<p>paragraph</p>`

## Automation Workflow for Claude Code

### Recommended Flow (every 3 days)

```
1. Read next pending post from content calendar
2. Web search for current facts related to the topic
3. Write full post following the 9-step SEO framework
4. Convert markdown to HTML
5. Insert into Supabase posts table as status: "draft"
6. Commit any associated files to GitHub if needed
7. Send notification to Nadim (email or Supabase flag) with:
   - Post title
   - Direct link to review in admin UI
   - Scheduled publish date

2. Save to /content/blog/[slug].mdx

5. Vercel auto-deploys
```

## Scheduling Options for Claude Code

### Option A — GitHub Actions (recommended)
```yaml
# .github/workflows/blog-publisher.yml
name: Blog Publisher
on:
  schedule:
    - cron: '0 15 */3 * *'  # Every 3 days at 8am MT (UTC-7)
  workflow_dispatch:

jobs:
  publish:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Run blog publisher
        run: python3 scripts/blog_publisher.py
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
          SUPABASE_URL: ${{ secrets.SUPABASE_URL }}
          SUPABASE_KEY: ${{ secrets.SUPABASE_KEY }}
          NOTIFY_EMAIL: ${{ secrets.NOTIFY_EMAIL }}
```

### Option B — Vercel Cron Jobs
If hosting on Vercel, use Vercel Cron to trigger a Next.js API route every 3 days that calls the Claude API and inserts into Supabase.

```json
// vercel.json
{
  "crons": [{
    "path": "/api/blog/generate",
    "schedule": "0 15 */3 * *"
  }]
}
```
